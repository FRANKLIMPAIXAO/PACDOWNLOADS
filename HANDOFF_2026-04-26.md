## Bloco 3 — Análise: banco e VPS Hostinger

### Estimativa de tamanho

Cada documento ocupa ~3 KB no Postgres + 8 KB médio no disco (XML).

| Cenário | Empresas | Docs/mês | Total/ano | 5 anos (SPED) |
|---|---|---|---|---|
| Pequeno | ≤10 | ~5.000 | ~700 MB | **~3,5 GB** |
| Médio | 10–50 | ~50.000 | ~7 GB | **~35 GB** |
| Grande | 50–200 | ~400.000 | ~52 GB | **~260 GB** |

Para descobrir o cenário real: pegar 1 empresa típica, contar docs/mês via SPED, multiplicar.

### VPS Hostinger + EasyPanel — viável

**EasyPanel** é painel Docker gerenciado (igual Coolify/Dokploy). Roda Docker Compose, templates oficiais para Postgres/Redis, SSL via Let's Encrypt, volumes persistentes.

**Services a configurar:** backend FastAPI, Celery worker, Celery beat, Postgres, Redis, frontend Next.js. RAM ociosa ~1,5 GB; picos 3–4 GB.

| Cenário | Plano Hostinger | Specs | Preço aprox |
|---|---|---|---|
| Pequeno | **KVM 2** | 2 vCPU, 8 GB RAM, 100 GB NVMe | R$ 50–70/mês |
| Médio | KVM 4 | 4 vCPU, 16 GB RAM, 200 GB NVMe | R$ 90–130/mês |
| Grande | KVM 8 + S3 | 8 vCPU, 32 GB RAM + bucket S3 | R$ 200+/mês |

**Não usar KVM 1** (4 GB) — apertado para rodar tudo junto.

### Ajustes obrigatórios antes de subir
1. Criar `Dockerfile` para backend (Python 3.12 + deps) — **falta**.
2. `Dockerfile` ou `nixpacks.toml` para frontend Next.js — **falta**.
3. Trocar `DATABASE_URL` para Postgres do EasyPanel.
4. Volume persistente para `/app/storage/xmls`.
5. Vars de ambiente no painel: `SECRET_KEY` forte, `NUVEM_CLIENT_ID/SECRET`, `USE_MOCK_NUVEM_FISCAL=false`, `NEXT_PUBLIC_API_URL=https://api.seudominio.com`.
6. CORS atualizar `allow_origins` no `main.py` para o domínio público.
7. Backup automático Postgres + semanal de XMLs (Hostinger Object Storage ou Backblaze B2 ~US$ 6/TB/mês).
