## Bloco 3 — Movimentação para fora do OneDrive

### O problema

OneDrive sincroniza arquivos em background. Quando Next.js (Node) tenta ler `node_modules/react/jsx-runtime` durante render, OneDrive pode estar segurando o arquivo → erro `UNKNOWN: unknown error, read`. Isso quebrava 3 das 7 rotas no smoke HTTP, mesmo com type-check passando e código correto.

### A solução

Movido o projeto via `robocopy /MOVE` para `C:\dev\pac-xml-downloader`:

```bash
robocopy "C:\Users\pacpa\OneDrive\...\pac-xml-downloader" "C:\dev\pac-xml-downloader" /E /MOVE
```

Foi necessário matar 5 processos `node.exe` zumbis antes (`taskkill /F /PID ...`).

### Resultado

Smoke test ponta-a-ponta no novo path: **todas 7 rotas com 200 OK**:

```
=== / ===            200 (16.3s — primeira compilação)
=== /login ===       200 (2.6s)
=== /empresas ===    200 (0.8s)
=== /empresas/novo ===  200 (0.8s)
=== /empresas/1 ===  200 (3.0s)
=== /documentos ===  200 (0.8s)
=== /relatorios ===  200 (0.7s)
```

**Comparação**: no path antigo (OneDrive) `/empresas/1`, `/documentos`, `/relatorios` retornavam 500 com erro de leitura em `node_modules`. No novo path, 200 limpo.

## Para sair do mock

1. No painel da Focus NFe (https://app.focusnfe.com.br/), pegar o token-mestre da conta.
2. `.env`:
   ```
   USE_MOCK_FOCUS_NFE=false
   FOCUS_AMBIENTE=homologacao   # comece em homolog!
   FOCUS_MASTER_TOKEN=<token-da-conta>
   ```
3. No frontend `/empresas/{id}` → "Cadastrar empresa na Focus" → preencher endereço completo + upload .pfx + senha → enviar. O backend chama `POST /v2/empresas` com o master token, recebe o token específico daquela empresa, e salva criptografado em `empresas.focus_token`.
4. **OU** se já cadastrou no painel manualmente: usar **"Importar token Focus"** com o token de homolog gerado.
5. Executar robô de distribuição para janela de 7 dias retroativos.
